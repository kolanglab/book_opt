# 最適化入門

**同じ計算を、もっと速く、もっと小さく。その技術には 60 年分の蓄積がある。**

`a * 2` を見て `a + a` や `a << 1` に書き換える。
ループの外に出せる計算を外に出す。
何度も呼ぶ小さなメソッドを呼び出し元に展開する。
動的言語で「このメソッドは前回 `Integer` だった」と覚えておいて分岐を消す——。

言語処理系の**最適化**（optimization）とは、
プログラムの**意味を変えずに**、実行時間・メモリ・コードサイズといった
コストを下げる変換のことです。本書は、古今東西の言語処理系が使ってきた
最適化技法を、静的言語・動的言語の区別なく、**ピープホール最適化から
全プログラム解析（whole program analysis）まで**、とにかく幅広く集めた地図です。

対象読者は、プログラミングをひととおり書けて、これから言語処理系の中身を
学ぼうとしている学部 3 年生くらいの初学者です。コンパイラやインタプリタを
作ったことがなくても読めるように、用語はその都度説明します。

## 本書の方針

- **「こういう最適化が知られている」で流さない。** すべての技法について、
  なぜ正しいのか・どう実装するのかを、動く Ruby のコードと小さな例で示します。
- **一次文献を引く。** 本文中の上付き \[著者, 年\] は巻末の参考文献へのリンクです。
  最適化は研究の歴史が長く、1970 年代の論文から最新の arXiv プレプリントまで、
  原文を読むのが理解の近道です。
- **静的も動的も。** Fortran/C 系の古典的データフロー最適化も、
  Smalltalk/Self/JavaScript/Ruby 系の動的最適化も、同じ本の中で扱います。

## 本書の構成

- **導入** — 最適化とは何か、何を守り（正当性）、何と引き換える（コスト）のか。
- **第 I 部「最適化の土台」** — [中間表現](intermediate-representation.md)（IR・基本ブロック・CFG・SSA）と、
  [データフロー解析](dataflow-analysis.md)（格子・不動点・抽象解釈）。すべての最適化が立つ地面。
- **第 II 部「古典的・局所的最適化」** — [ピープホール](peephole.md)、
  [定数畳み込みと値番号付け](local-optimization.md)、[不要コード除去](dead-code-elimination.md)、
  [SSA 上の最適化](ssa-optimization.md)、[ループ最適化](loop-optimization.md)。
- **第 III 部「手続き間と全プログラム最適化」** — [インライン展開](inlining.md)、
  [手続き間解析](interprocedural-analysis.md)（別名・エスケープ解析）、
  [全プログラム最適化](whole-program-optimization.md)（脱仮想化・LTO・部分評価）。
- **第 IV 部「動的言語の最適化」** — [動的ディスパッチとインラインキャッシュ](dynamic-dispatch.md)、
  [投機的最適化](speculative-optimization.md)（型特殊化・脱最適化・トレーシング・BBV）。
- **第 V 部「コード生成にまつわる最適化」** — [命令選択](instruction-selection.md)、
  [レジスタ割り当て](register-allocation.md)、[命令スケジューリング](instruction-scheduling.md)。
- **第 VI 部「最前線」** — [equality saturation](equality-saturation.md)、
  [superoptimization と検証](superoptimization.md)、[機械学習による最適化と未来](ml-and-future.md)。
- **付録** — [用語集](glossary.md)。
