# 古典的・局所的最適化

第 II 部は、コンパイラ最適化の「定番メニュー」を一つずつ料理する。
これらは 1960〜90 年代にほぼ確立した古典だが、今日の LLVM[](#cite:lattner2004) や
各言語の JIT でも中核として動き続けている、現役の技術である。

進み方は「狭い視野から広い視野へ」だ。
まず[ピープホールの章](peephole.md)で、数命令だけを覗いて書き換える最も局所的な最適化を見る。
次に[局所最適化の章](local-optimization.md)で、定数畳み込み・定数伝播・コピー伝播・
共通部分式除去・値番号付けといった、基本ブロック内の定番変換をそろえる。
[不要コード除去の章](dead-code-elimination.md)では、「もう要らない計算」を安全に消す。

そして[SSA 最適化の章](ssa-optimization.md)で視野を関数全体に広げ、
SSA 形式が局所最適化をいかに大域最適化へと引き上げるか
（疎な条件付き定数伝播（SCCP）[](#cite:wegman1991)や大域的値番号付け[](#cite:alpern1988)）を見る。
最後に[ループ最適化の章](loop-optimization.md)で、実行時間の大半を占めるループに狙いを定め、
ループ不変式の移動・強度低減・アンローリング・ベクトル化を扱う。

各最適化について、本書は「何を仮定すれば正しいか」を必ず述べる。
最適化はプログラムの意味を変えてはならない。この制約こそが、すべての技法の設計を決めている。
